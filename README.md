# 福本屋サラダ館｜店舗間在庫移動・外部注文依頼管理

複数店舗（矢掛本店・倉敷店・総社店・井原店）を持つ福本屋サラダ館向けの業務アプリ。
店舗間の在庫移動依頼と本店への外部注文依頼を一元管理し、未処理タスクと通知をスマホからすぐ確認できます。

「洗練SaaS型」デザイン言語（Linear / Stripe 系）でフル再構築したバージョン。

## 技術スタック

- **Vite + React 18 + TypeScript**
- **Tailwind CSS**（CSSカスタムプロパティによるライト/ダークの2テーマ）
- **Framer Motion**（ページ遷移・カードのスタッガー・カウントアップ・トースト・モーダル）
- **lucide-react**（アイコン）
- データは **localStorage** に保存（プロトタイプ。バックエンド差し替え前提の構造）

## 開発

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # 型チェック + 本番ビルド
npm run preview  # 本番ビルドのプレビュー
```

## デプロイ（Vercel）

静的SPA。`vercel.json` にセキュリティヘッダ（CSP / X-Frame-Options / X-Content-Type-Options /
Referrer-Policy / Permissions-Policy）と長期キャッシュを同梱済み。

```bash
vercel --prod
```

## 旧版レビューで検出した課題と本版での対応

| 課題（旧版） | 本版での対応 |
|---|---|
| 従業員削除に確認なし＝誤削除リスク | 削除前に確認ダイアログ（ボトムシート）を表示 |
| 依頼者に他店従業員も選べる（整合崩れ） | 依頼者は依頼元店舗の従業員のみ／担当者は依頼先店舗の従業員のみ |
| 他店の従業員も削除できる（権限境界なし） | 自店のみ編集可。他店は「他店」表示で読み取り専用 |
| コントラスト不足 3 件（WCAG AA未達） | ライト/ダーク両モードで全画面 AA 達成（axe 0 件） |
| バリデーションが赤枠なし・読み上げなし | 赤枠＋リング＋インラインメッセージ＋最初のエラーへ自動フォーカス＋aria-live |
| favicon 404／console エラー | favicon 設置・console クリーン |
| ログイン画面に main ランドマークなし | 全画面に適切なランドマーク＋スキップリンク |
| セキュリティヘッダ HSTS のみ | vercel.json で主要ヘッダを追加 |
| 通知パネルが遷移後も開いたまま | 外側クリック / Esc / 遷移で自動クローズ |
| デスクトップで余白過多 | デスクトップ＝サイドバー、モバイル＝ボトムナビの最適レイアウト |
| （新規）ダークモードなし | ライト/ダーク切替＋端末設定追従＋永続化 |

## 画面構成

- **ログイン**：店舗選択
- **ホーム**：未処理アラート、ステータス集計（カウントアップ）、最新の依頼
- **新規依頼**：種別（店舗間/外部注文）切替、入力フォーム＋バリデーション
- **依頼一覧**：検索＋ステータス絞り込み
- **従業員**：自店の追加・削除（確認付き）、全店の一覧
